# Release Notes

## A New Look

- Stimma has been redrawn end to end in a new design language: quieter surfaces, hairline separators instead of boxes-in-boxes, one consistent set of type, color, and motion vocabulary across every screen
- Browsers, boards, and the library are now contact sheets — borderless tiles on a matte with even gutters and a single selection color
- Home and Project overviews have been cleaned up and streamlined
- Dialogs, menus, dropdowns, and progress indicators all run on shared components, so they open, animate, and stack the same way everywhere

## Chat

- Chat artifacts: the agent can stage a deliverable in the chat and revise it in place, so a conversation ends with one finished asset instead of creating a mess of slightly different assets.
- Fixed context management bugs related to multimodal conversations


## Audio & Voice

- Stimma Account now supports ElevenLabs tools for voice, music, and SFX
- Provider-neutral voice picker with more room to browse

## Flows

- Send to Flow: hand an asset from anywhere in the app straight into a flow
- Tools that are blocked or unavailable now say so in the flow instead of stalling silently
- Fixed profile isolation for flow programs and run state, preventing cleanup in one profile from touching another

## Library & Assets

- New "Recently imported" sort in the browser
- Bulk tagging moves from a modal to an instant-apply tag picker popover
- The info panel is rebuilt around the asset's action row, lineage timeline, and version list
- The media context menu is now grouped by intent — mark, organize, act, explore, output, with trash last
- Tool filters name the tool an asset actually came from, even when that tool has since been renamed, retired, or disconnected
- Empty, unnamed boards, flows, and chats left behind by a misclick are cleaned up automatically after 12 hours; anything you named, edited, or added to is never touched

## Speed

- Thumbnails now load under end-to-end admission control: nearest-to-viewport first, paused during fast scrolling, and never generated for tiles that scrolled away — long scrolls and opening the slideshow no longer hose the server
- Thumbnail generation splits light and heavy work into separate pools and coalesces duplicate requests
- Emptying a large trash is dramatically faster and the app remains responsive while it runs

## Deletion & Source Folders

- Emptying the trash now deletes files that live in your Source folders, not just Stimma's copies
- Source folders keep their identity when you move them on disk
- macOS file-permission denials are reported plainly instead of surfacing as generic errors
- Fixed deletion progress and time-remaining estimates

## Fixes & Polish

- Fixed the slideshow control bar colliding with video transport controls
- Fixed the single-image drop well disappearing on flow input forms
- Fixed stale filter options lingering after the underlying data changed
- Fixed cloud provider state when signed out
- Fixed vision capability detection for some local models
- Fixed the tool preset baseline so presets no longer read as modified right after applying
